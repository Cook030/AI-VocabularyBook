## 3. 项目简介

### 项目简介

AI 智能单词本系统是一个基于 AI 的个性化英语单词学习平台。用户可以搜索英文单词，获取 AI 生成的释义、例句和同义词，并将单词加入个人生词本进行学习管理。

### 核心功能

- **AI 单词查询**：调用阿里云百炼 API，实时生成单词释义、例句和同义词
- **用户认证**：支持用户注册、登录，使用 JWT Token 进行身份验证
- **生词本管理**：用户可以添加、查看、标记掌握状态、移除单词
- **响应式前端**：基于 React + Vite 构建，支持明暗主题切换

### 技术栈

| 层级    | 技术                                          |
| ----- | ------------------------------------------- |
| 前端    | React 18 + TypeScript + Vite + React Router |
| 后端    | Go + Gin + GORM                             |
| 数据库   | MySQL 8.0                                   |
| AI 服务 | 阿里云模型服务                                     |
| 容器化   | Docker + Docker Compose                     |
| 反向代理  | Nginx                                       |

## 4. 运行指南

### 前置依赖

- **Docker** (>= 20.10)
- **Docker Compose** (>= 2.0)

### 配置环境变量

在 `backend/` 目录下创建 `.env` 文件：

```env
# backend/.env 文件
DATABASE_URL=root:rootpassword@tcp(db:3306)/ai_vocabulary_db?charset=utf8mb4&parseTime=True&loc=Local
AI_API_KEY=your_ai_api_key
JWT_SECRET=your_jwt_secret_key_here
```

### 一键启动

```bash
# 在项目根目录执行
docker-compose up -d --build
```

### 验证服务状态

```bash
docker-compose ps
```

所有服务状态应为 `Up` 且 `healthy`（如果配置了健康检查）。

### 停止服务

```bash
docker-compose down
```

如需清除数据（重置数据库）：

```bash
docker-compose down -v
```

### 访问地址

| 服务     | 地址                     | 说明                     |
| ------ | ---------------------- | ---------------------- |
| 前端页面   | <http://localhost>     | Web 应用入口               |
| 后端 API | <http://localhost/api> | API 根路径（通过 Nginx 反向代理） |

### 环境变量说明

| 变量名            | 必填 | 说明            | 示例                                                |
| -------------- | -- | ------------- | ------------------------------------------------- |
| `DATABASE_URL` | 是  | 数据库连接字符串      | `root:password@tcp(db:3306)/ai_vocabulary_db?...` |
| `AI_API_KEY`   | 是  | 百炼控制台 API Key | `sk-xxxxxxxxxx`                                   |
| `JWT_SECRET`   | 是  | JWT 签名密钥      | `your_secret_key`                                 |
| `TZ`           | 否  | 时区设置          | `Asia/Shanghai`                                   |

### 接口概览

| 接口    | 方法     | 路径                               | 认证 | 说明            |
| ----- | ------ | -------------------------------- | -- | ------------- |
| 注册    | POST   | `/api/auth/register`             | ❌  | 用户注册          |
| 登录    | POST   | `/api/auth/login`                | ❌  | 用户登录，获取 Token |
| 搜索单词  | GET    | `/api/words/search`              | ✅  | AI 查询单词释义     |
| 生词本列表 | GET    | `/api/user-words`                | ✅  | 获取用户生词本       |
| 添加单词  | POST   | `/api/user-words`                | ✅  | 添加单词到生词本      |
| 更新状态  | PATCH  | `/api/user-words/:wordID/status` | ✅  | 标记掌握状态        |
| 移除单词  | DELETE | `/api/user-words/:wordID`        | ✅  | 从生词本移除        |

详细接口文档请参阅 [docs/api.md](docs/api.md)。

### 数据库设计

系统包含三张核心数据表：

| 表名           | 说明                        |
| ------------ | ------------------------- |
| `users`      | 用户表：存储用户账号密码              |
| `words`      | 单词表：存储单词及 AI 生成的释义、例句、同义词 |
| `user_words` | 关联表：用户与单词的多对多关系及掌握状态      |

详细数据库设计请参阅 [docs/db.md](docs/db.md)。
